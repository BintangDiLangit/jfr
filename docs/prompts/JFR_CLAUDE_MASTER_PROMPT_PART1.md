## CODING RULES

Gunakan reusable component.

Jangan duplicate code.

Gunakan Server Components bila memungkinkan.

Gunakan Client Components hanya jika dibutuhkan.

Strict TypeScript.

Clean Architecture.

Jangan membuat file besar >300 baris jika bisa dipecah.

Selalu gunakan best practice.

Sebelum mulai coding:

1. Analisis requirement.
2. Buat PRD.
3. Buat struktur folder.
4. Buat Firestore schema.
5. Buat wireframe.
6. Baru implementasi.
